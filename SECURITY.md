# Security Policy

## Supported versions

Security fixes are applied to the latest released version.

## Reporting a vulnerability

Do not open a public issue containing exploit details, credentials, tokens or
personal data.

Use GitHub private vulnerability reporting from the repository **Security**
tab when it is enabled. If private reporting is not available, open a minimal
public issue asking the maintainer to establish a private contact channel.
Do not include technical details in that issue.

## Credential exposure

If a credential was committed or posted publicly:

1. revoke or rotate it immediately at the platform,
2. remove it from the current branch,
3. consider the complete Git history exposed,
4. notify affected users when applicable.

Deleting a secret from the latest commit does not make the old secret safe.
